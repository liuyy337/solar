### Alignment

在对太阳观测数据进行科学分析之前，首先需要对原始数据进行基础的数据处理。望远镜科学团队需要将探测器接收到的原始数据(Level 0)进行暗场扣除(消除仪器自身的热噪声)和平场校正(消除探测器各个像素感光不均匀的影响)，从而发布可供研究者直接使用的科学数据(Level 1)。然而，由于仪器的微小抖动、太阳的自转，以及不同望远镜的视场差异，Level 1 图像在时间序列上存在抖动，在空间坐标上存在错位。为了保证我们能在多波段联合分析下精准追踪太阳大气的演化，必须对图像进行严格的对齐(Alignment)。图像对齐主要包含以下两个关键步骤：

**1 单波段内的时间序列对齐**

第一步是消除同一波段图像在时间序列上的相对漂移。当我们连续观测一个太阳活动区时，随着时间推移，目标区域不仅会随着太阳自转而移动，望远镜本身也会有不可避免的视轴抖动。为了解决这个问题，我们通常采用光流法(Optical Flow)。简单来说，光流法是一种计算机视觉技术，它通过比较时间序列中相邻两帧图像的像素灰度变化，自动追踪图像中特征结构(如黑子、米粒组织等)的运动轨迹，计算出每个像素的移动方向和距离。得到这个位移场后，程序会自动将后一帧图像向相反方向平移相应的距离，从而抵消抖动和宏观位移。经过光流法处理后，同一个物理结构在整个视频序列中将稳定保持在同一位置，为后续的动力学分析提供稳定的参照。

**2 跨波段与跨仪器的空间对齐**

第二步是解决不同波段、不同观测仪器之间的空间坐标错位。例如，地基望远镜(GST)和空间卫星(SDO)即便在同一时刻拍摄同一个太阳活动区，由于它们的观测视角、分辨率和坐标系定义不同，图像直接叠加是无法吻合的。跨波段对齐的核心思想是寻找共同特征并统一世界坐标系。具体操作步骤如下：

1. 确立参考基准： 首先，需要选择一张具有明显结构特征的图像作为绝对参考。例如，选取SDO/HMI在特定时间的连续谱图像作为全局参考标准 。
2. 特征匹配与偏差计算：在待对齐的图像(如NVST的TiO波段)中，寻找与参考图像相对应的共同物理结构(例如连续谱图像和TiO图像中都存在的黑子本影)。通过比对这些共同特征，计算出两张图像在水平轴(X)和垂直轴(Y)上的偏移量。
3. 更新 FITS 头文件：头文件(Header)中记录了图像的坐标信息。我们只需将计算出的X和Y方向的偏差值，分别补偿到该图像FITS头文件的 `CRVAL1` (X轴参考坐标)和 `CRVAL2` (Y轴参考坐标)中，就实现了统一坐标系。

### Example

`preview.py`输出的头文件信息:
```
('HISTORY', 'Photometric flux ratio: 0.7440', '')
('HISTORY', 'Aligned relative to rolling anchor. Cum_Shift: X=-1.15, Y=-18.44', '')
('HISTORY', 'Manual align residual shift applied: Y=65.0, X=49.0', '')
('HISTORY', 'Aligned to bottom sunspot. Cum_Shift: X=-0.66, Y=-4.35', '')
```

头文件里的这些信息说明此数据已经做了序列间的滚动参考对齐(rolling anchor)和以底部太阳黑子(bottom sunspot)作为锚点的精细对齐，第一步已经完成了。但是多波段之间的对齐，也就是`CRVAL1`和`CRVAL2`的修改还没有做，我们需要把它们的图像放到一起，然后手动调整`CRVAL1`和`CRVAL2`的值。

`overview.py`生成的`overview.png`展示了三个波段的概览,图像下面的3张是它们的数据分布直方图,根据直方图我们确认了`norm=colors.Normalize(10000,13000)`的取值范围,这个值必须覆盖数据点的主要范围,这样图像才能显示清楚.

`alignment.py`绘制了3个波段的子图,我们将根据这个图来手动调整`CRVAL1`和`CRVAL2`.这里的fits文件里没有`CRVAL`的值,所以无法使用`sunpy.map.Map()`来读取。我们使用`fits.open()`读取数据`data`之后,随便给了它一对`CRVAL`的值,也就是:
```
x1, y1 = 0, 0
x2, y2 = 0, 0
x3, y3 = 0, 0
```
然后我们使用自定义的`create_map`函数，把`data`打包成了一个sunpy的map对象，map自带WCS坐标系。
```
    bottom_left = SkyCoord((xc - dx/2) * u.arcsec, (yc - dy/2) * u.arcsec, frame=map1.coordinate_frame)
    top_right = SkyCoord((xc + dx/2) * u.arcsec, (yc + dy/2) * u.arcsec, frame=map1.coordinate_frame)
    map1 = map1.submap(bottom_left, top_right=top_right)
    map2 = map2.submap(bottom_left, top_right=top_right)
    map3 = map3.submap(bottom_left, top_right=top_right)
```
我们马上按照`xc, yc, dx, dy`对生成的map进行了裁剪，然后绘制了这个小区域(这个小区域内有一个黑子，可以作为参考特征)内的多波段图像。这时候我们可以看到三张子图里面的黑子的位置是不一致的，这时候我们需要手动微调`x1, y1, x2, y2, x3, y3`的值，让三张子图的黑子对齐，最后的调整结果如下：
```
x1, y1 = -1.5, -4.5
x2, y2 = -6.8, 0.3
x3, y3 = -10.7, -0.5
```

这就是我们需要的偏移值，我们使用`write_fits.py`把这些值写入，就得到了新的对齐了的fits文件。(这里的偏移只是这三个波段之间的相对偏移，根据现在的`x,y`值来改`CRVAL`虽然对齐了这3个波段，但是其坐标并不是真实的世界坐标！要想得到真实的坐标，需要再和SDO/HMI上的黑子结构来对齐。开始我们的`x, y`直接取的0, 实际上应该根据HMI的图像来取一个真实的坐标值)

最后使用`multiplot.py`，我们读取新生成的fits文件，并且裁剪子图，制作视频。
